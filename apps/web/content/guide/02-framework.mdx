# The Framework

Skills connect through a product development lifecycle. Each phase has its own focus and outputs.

## The Four Phases

```
Discover → Define → Design → Deliver
```

### Discover

Understand the problem before jumping to solutions.

**Skills:** User research methods, stakeholder interviews, problem framing

**Outputs:** Clear problem statement, user needs identified, constraints mapped

### Define

Shape the solution with testable specifications.

**Skills:** `validate-stories`, acceptance criteria frameworks

**Outputs:** User stories, acceptance criteria, technical feasibility assessment

### Design

Align stakeholders and get buy-in.

**Skills:** `make-deck`, presentation frameworks, stakeholder communication

**Outputs:** Aligned team, clear vision, agreed priorities

### Deliver

Ship with quality and visibility.

**Skills:** Delivery monitoring, quality gates, risk flagging

**Outputs:** Working software, predictable delivery, satisfied stakeholders

## How Phases Connect

```
Problem discovered
    ↓
Problem defined with specs
    ↓
Solution designed and aligned
    ↓
Solution delivered with quality
    ↓
Feedback loops back to Discover
```

Not linear — real work has back-and-forth. But each phase has a clear focus.

## Choosing Your Entry Point

| Where you are                  | Start here |
| ------------------------------ | ---------- |
| Unsure about the problem       | Discover   |
| Problem clear, need specs      | Define     |
| Solution clear, need alignment | Design     |
| Aligned, need to ship          | Deliver    |

## The Philosophy

Every product decision is a bet. Lensed Build helps you make better bets by:

1. **Validating early** — Learn before you build
2. **Communicating clearly** — Align before you commit
3. **Delivering reliably** — Ship what you promised
