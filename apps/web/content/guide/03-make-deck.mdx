# make-deck

Build a persuasive presentation deck with beginning/middle/end structure. Use when you want to create slides, build a presentation, or craft a pitch deck.

## When to Use

- Pitching an idea to stakeholders
- Presenting research findings
- Communicating a vision
- Aligning a team on direction

## The Structure

Every deck has three parts:

- **Beginning** — The setup. Why should the audience pay attention?
- **Middle** — The main points. Build tension with ups and downs.
- **End** — Summary. Clear next steps.

## Slide Format

Each content slide follows this pattern:

```
[but/therefore]
[Slide N]

Headline text

Optional lead line

- point 1
- point 2
- diagram suggestion (optional)
```

Title slides anchor each section:

```
# Section Title

Supporting line
```

## Transitions

Link slides with **BUT** or **THEREFORE** only.

- **BUT** — Introduces contrast, complication, or a new angle
- **THEREFORE** — Introduces a consequence, solution, or conclusion

Never use "and" or "because" to connect slides.

## The Workflow

1. Ask about the presentation goal, audience, and key message
2. If the core value is unclear, ask "why?" until it's sharp
3. Build the deck following the format
4. Ask the user to review and iterate

## Example

```
# The Problem

We spend 40% of our time on work we shouldn't have to do.

---

but

# Slide 1

You Already Think Like a Programmer

Grasshopper. Dynamo. Excel. You've been thinking in logic.

---

therefore

# Slide 2

Code Is Just Automation With a Text Editor

Python reads files, renames layers, extracts data.
```

## Safe Assumptions

- The audience is initially uninterested
- They have little or no prior knowledge of the topic
- Keep decks short — if it can't be condensed, it's not ready
