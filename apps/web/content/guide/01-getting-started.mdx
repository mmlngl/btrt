# Getting Started

Lensed Build skills are AI agent workflows. Use them individually or as a framework for building with the right lenses.

## How It Works

Each skill is a structured workflow. Load the skill, provide your context, and follow the steps.

**The key principle:** No need to use everything at once. Start with one skill that matches your current phase.

## Why Skills?

Traditional advice is either:

- **Theoretical** — Sounds good, doesn't translate to action
- **Tactical** — Here's a template, fill in the blanks

Skills bridge both. They're:

- **Actionable** — Step-by-step workflows you can follow
- **Flexible** — Work standalone or chained together
- **Reusable** — Apply the same skill to different problems

## Your First Skill

Start with `make-deck` if you need to communicate an idea. Start with `validate-stories` if you're managing development.

Don't try to use all skills at once. Build confidence with one, then expand.

## Framework vs. Standalone

You can use skills in two ways:

**Standalone** — Pick the skill that matches your immediate need.

**Framework** — Use multiple skills in sequence for a complete process.

> The framework approach works best when you're building something from scratch or doing a major pivot.
